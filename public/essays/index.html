Q1. Explain the difference between HTML, CSS and JS.

  • HTML: Content: describes the structure of a web page, and tells the browser how to display the content. It is a markup language that uses tags to define elements within a document. Most elements require a beginning and end tag, with the content placed between the tags. It is human-readable, meaning markup files contain standard words, rather than typical programming syntax.
  • CSS: Appearence: defines styles for web pages, including the design, layout, and variations in display for different devices and screen sizes.
  • JavaScript: Behavior: It is a programming language that adds interactivity to your website. This happens in games, in the behavior of responses when buttons are pressed or with data entry on forms; with dynamic styling; with animation, etc. In JavaScript, it is possible to change the type of a variable or add new properties or methods to an object while the program is running.

What are Single Page Apps (SPAs)? How do they differ from traditional multi-page Web Sites?

  • A single-page application helps keep the user in one web page that you visit and doesn’t require page reloading during use, and works within a browser. Its content is presented to the user in a simple, easy, and “natural” environment in the browser, which means no page reloads, no extra wait time. Example: Gmail, Google Maps, Facebook, GitHub.
  • A Multiple-page application works in a “traditional” way that loads new pages when you click a link. MPAs provide a visual map of where to go with few level menu navigations. MPAs are large, bigger than SPAs as SPA can cache any local storage effectively.

What is the difference between Web Designer, Front End Developer and Back End Developer?

  • A Web Designer could be someone who designs the look and feel of a site and will never touch the code. Specifically, UX designers study and research how people use the sites while UI designers are generally focused on visual design, but they might know light HTML, CSS, and JS so they can communicate their ideas more effectively to the front end developer.
  • A Front End Developer transfers the designs to what the user can see and interact with. It includes everything the user experiences directly: from text and colors to buttons, images, and navigation menus. It’s important to note that, although frontend developers deal with the visual and interactive side of a website, they don’t actually design these front-facing elements. Main front end language: HTML, CSS, JS.
  • A Back End Developer works on the server side and builds Web Services, APIS, Storage Secure Info. The backend usually consists of three parts: a server, an application, and a database. Users can’t see the backend work, but this code is what communicates the database information to the browser, which enables the user-facing side to exist.

Distinguish the difference between Site Relative, Document Relative, and Absolute Paths

  • Site Relative: Start with forward slash '/path/to/file.jpg', they're more flexible. Document Relative: Relative to the file you are working on; start with ./ They're particularly useful when the current site/document and the linked site/document are in the same folder and are likely to remain together; Absolute Paths: An absolute path always contains the root element and the complete directory list required to locate the file.
  • Absolute paths always provide the complete website address where you want the user to go, whereas relative links only point to a file or a file path. When a user clicks a relative link, the browser takes them to that location on the current site. For that reason, you can only use relative links when linking to pages or files within your site, and you must use absolute links if you're linking to a location on another website.

What does it mean for a company to be Agile. What is the purpose of using Agile, and its Pros and Cons.

In essence, an agile approach involves breaking one task into discrete modules, which they test individually before continuing. An agile team also practice the following values to the entire organization:

  1. Individuals and interactions over process and tools
  2. Working software over comprehensive documentation
  3. Customer collaboration over contract negotiation
  4. Responding to change over following a plan
  • Pros: Rapid delivery; Adaptable; Collaborative (better communication); Quick problem detection; Continuous improvement.
  • Cons: Hard to predict; Ephemeral planning; Paperwork gets left behind; Over focused on the client-side; Lack of overall cohesion; Challenges at scale.
